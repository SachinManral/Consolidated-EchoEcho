What is Echo Echo?

Echo Echo is an AI-powered songwriting inspiration tool. You describe what you want to feel — a mood, a genre, a theme, a tempo — and Echo Echo generates a short original instrumental sketch, writes matching lyrics, and checks whether the lyrics are copyright-safe. The result is a complete creative starting point you can download and build on.

It's built for songwriters who want a spark, content creators who need royalty-free background music, and developers who want to integrate AI music generation into their own apps.

How It Works

When you press Generate, here's the full pipeline that runs:

Song record saved → returned to client
  1. You submit a generation requestChoose mood, genre, theme, instruments, tempo, and energy. These get compiled into an AI-ready text prompt by the backend's composer.py.
  2. Request is sent to Kie.AI (Suno v4)The backend calls Kie.AI's API with your prompt. Kie.AI queues a Suno v4 music generation job and posts a callback to Echo Echo when done. The server polls for completion in parallel.
  3. Audio is downloaded and savedOnce Kie.AI confirms the track is ready, Echo Echo downloads the MP3 to backend/generated/ and stores a record in song_history.json.
  4. Lyrics are written by a CrewAI agentA Groq-powered lyricist agent (Llama 3.1 8B) reads your original prompt and composes a verse/chorus structure that fits the mood and style.
  5. Copyright check runs automaticallyA second CrewAI agent checks the generated lyrics against known songs for similarity, returning a risk score and safety status.
  6. Full result is returnedThe response includes the audio URL, download link, full lyrics, and copyright report — all in one API response.

Getting Started

Echo Echo has two ways to access it: through the web app (just open the URL in your browser) or the REST API (for developers building their own integrations).

Using the Web App

  1. Open the appGo to https://consolidated-echoecho.onrender.com in any modern browser. No install required.
  2. Create an accountClick Get started on the landing page. Sign up with your email and a password. (If sign-ups are closed, you need an invite code from the admin.)

Generating Music

Echo Echo supports two generation modes. The app automatically picks the best one based on your setup.

API Mode (Recommended)

Uses Kie.AI to call Suno v4 — the same AI that powers suno.com. Results are high-quality, full-length MP3 tracks (usually 2–4 minutes) delivered in ~30–90 seconds. Requires a KIEAI_API_KEY.

✅

This is what the deployed app uses. The cloud Render deployment always generates via API mode. MusicGen local mode is only available when running Echo Echo on your own machine with the heavy dependencies installed.

Local MusicGen Mode

Uses Facebook's open-source facebook/musicgen-small model running locally. No API key needed, but requires torch and transformers installed and a machine with enough RAM (4+ GB). Generation takes 1–6 minutes and produces a WAV file.

⚠️

MusicGen is excluded from the cloud deploy (requirements-deploy.txt omits torch/transformers) because the model is too large for free-tier servers. Use API mode in production.

Generation Flow

  1. Open DashboardGo to /dashboard.html after logging in.
  2. Set your parametersChoose mood(s), genre(s), theme(s), instruments, BPM (60–160), energy (1–10), and complexity. Optionally type a custom prompt for finer control.
  3. Click "Create Inspiration"The app sends a POST /api/generate request and begins polling GET /generation-status every second to show a live progress bar.
  4. Wait for generationAPI mode typically completes in 30–90 seconds. The progress bar cycles through stage names: Submitting, Analyzing mood, Composing melody, Writing lyrics, Checking copyright…
  5. Review your songYou're redirected to /output.html with the audio player, lyrics panel, copyright badge, and download buttons.

Song Parameters

All parameters are optional — Echo Echo picks sensible defaults if you skip them. Combining multiple values from the same field (e.g. two moods) produces a blended result.

ParameterTypeRange / OptionsDescription
mood / moodsstring / string[]Any emotionThe emotional tone. e.g. Sad, Romantic, Hopeful, Dreamy, Dark, Energetic.
theme / themesstring / string[]Any topicThe subject matter. e.g. Rain, Love, Night, Memories, Nature.
style / genresstring / string[]Any genreMusical style. e.g. Lo-fi, Jazz, Classical, Electronic, Ambient.
instrumentsstring[]Any instrument nameInstruments to feature. e.g. Piano, Flute, Guitar, Violin, Saxophone.
tempointeger60–160 BPMSpeed of the track. 60–80 = slow/ballad, 90–110 = mid, 120–160 = fast/dance. Default: 90.
energyinteger1–10Intensity. 1–3 = gentle/ambient, 4–6 = moderate, 7–10 = high energy. Default: 4.
complexitystringSimple · Moderate · RichArrangement density. Simple = sparse, Rich = layered and full. Default: Moderate.
durationinteger1–600 secondsRequested duration hint (API mode targets 30s clips; actual length depends on Suno). Default: 30.
custom_promptstringFree textOverride or supplement the auto-generated prompt with your own words. e.g. "rainy evening in Tokyo, nostalgic piano".
💡

Tip: The more specific you are, the better. Combining moods: ["Sad","Romantic"] + themes: ["Rain","Love"] + instruments: ["Piano","Guitar"] produces much more targeted results than a single mood.

Your Library

Every track you generate is saved to your library at /echo.html. The library persists on the server in song_history.json (on the free Render tier this resets on redeploy — use a Render Disk for permanent storage).

What you can do from the library

  • Play any track — inline audio player for every song
  • Download originals — full MP3 or WAV files
  • Trim to 30 seconds — cuts API-generated MP3s with FFmpeg for easy sharing
  • View lyrics — see the AI-generated lyrics for any track
  • Re-run copyright check — re-analyze lyrics against known songs at any time
  • Song IDs — each track has a unique 4-letter code (e.g. ABCD) used in download URLs

Lyrics & Copyright

Lyrics Generation

After music generation, Echo Echo automatically writes lyrics using a CrewAI Lyricist agent powered by Groq Llama 3.1 8B. The agent is given the song's mood, theme, genre, tempo, and instruments as context. It writes a verse/chorus structure designed to fit the feel of the track.

Lyrics are included in the /api/generate response and stored in the song record. If Groq is unavailable, the lyrics field gracefully falls back to a note explaining the service is unavailable — generation still succeeds.

Copyright Check

A second CrewAI agent evaluates the generated lyrics for similarity to known, copyrighted songs. It returns:

  • status — safe or risky
  • similarity_score — 0.0–1.0 confidence score
  • risk — None, Low, Medium, or High
  • notes — human-readable explanation from the agent
  • matches — list of potentially matching songs (title, artist, copyright status)
ℹ️

The copyright check is AI-generated and not a legal opinion. Use it as a creative guide, not a legal clearance. Always verify with a copyright professional before commercial use.

API Reference

All endpoints accept and return JSON. The base URL is the root of your deployment. No authentication header is required for generation endpoints in the current version — auth is enforced at the frontend layer.

🔗

Try all endpoints interactively in the Swagger UI ↗

POST /api/auth/login

Authenticate with email and password. Returns a bearer token to store in localStorage as echo_auth_token.

REQUEST{
  "email": "you@example.com",
  "password": "yourpassword"
}
RESPONSE 200{
  "ok": true,
  "token": "eyJhbGci...",
  "user": {
    "email": "you@example.com",
    "name": "Your Name"
  }
}

Returns 401 Unauthorized on wrong credentials.

POST /api/auth/signup

Create a new account. Requires ECHO_SIGNUPS_OPEN=1 environment variable to be set — otherwise returns 403.

REQUEST{
  "name": "Your Name",
  "email": "you@example.com",
  "password": "yourpassword"
}
RESPONSE 200{
  "ok": true,
  "token": "eyJhbGci...",
  "user": { ... }
}
⚠️

409 if email already exists. 403 if sign-ups are closed.

Environment Variables

Set these in a .env file locally or in the Render dashboard's Environment tab.

VariableRequiredDescription
GROQ_API_KEY required Groq API key. Powers lyrics generation and copyright checking via Llama 3.1 8B. Get one free at console.groq.com.
KIEAI_API_KEY required Kie.AI key for Suno v4 music generation. Without this, only local MusicGen mode is available (not suitable for cloud deploy). Get one at kie.ai.
PUBLIC_BASE_URL required The public URL of your deployed server (e.g. https://consolidated-echoecho.onrender.com). Kie.AI posts completion callbacks to this URL. Must be publicly reachable — localhost will not work.
KIEAI_CALLBACK_PATH optional Override the callback path. Default: /api/kieai/callback.
TRIM_DURATION_SECONDS optional Default duration when trimming audio. Default: 30.
ECHO_SIGNUPS_OPEN optional Set to 1 to allow public sign-ups. When unset or any other value, the signup endpoint returns 403 Forbidden (invite-only mode).
PYTHON_VERSION optional Python version for Render's build environment. Set to 3.12.6 in render.yaml.

Local Setup

Run Echo Echo on your own machine for development.

Prerequisites

  • Python 3.12+
  • Git
  • FFmpeg (optional — needed for audio trimming only)
  • A Groq API key (free at console.groq.com)
  • A Kie.AI API key (for API mode generation)
  • A public tunnel like ngrok or cloudflared for Kie.AI callbacks

Steps

# 1. Clone
git clone https://github.com/simplysandeepp/Consolidated-EchoEcho.git
cd Consolidated-EchoEcho

# 2. Create and activate virtual env
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create .env file
cat > .env <<EOF
GROQ_API_KEY=gsk_...
KIEAI_API_KEY=your-kieai-key
PUBLIC_BASE_URL=https://abc123.ngrok.io
EOF

# 5. Start the server
uvicorn backend.main:app --reload --port 8000

Open http://localhost:8000 in your browser. The Swagger UI is at http://localhost:8000/api/docs.

🔌

For Kie.AI callbacks locally: Start ngrok with ngrok http 8000, copy the HTTPS URL it gives you, and set that as PUBLIC_BASE_URL in your .env. Kie.AI must be able to POST to your machine for generation to complete.

Render Deployment

Echo Echo ships with a render.yaml that configures everything Render needs. Frontend and backend are a single service — no separate deploy.

Deploy in 4 Steps

  1. Connect your repo to RenderGo to render.com → New → Web Service → Connect GitHub → select simplysandeepp/Consolidated-EchoEcho.
  2. Render reads render.yaml automaticallyBuild command is set to pip install -r requirements-deploy.txt and start command to uvicorn backend.main:app --host 0.0.0.0 --port $PORT.
  3. Add environment variablesIn the Environment tab, add GROQ_API_KEY, KIEAI_API_KEY, and set PUBLIC_BASE_URL to https://consolidated-echoecho.onrender.com.
  4. Click DeployBuild takes ~2 minutes. Once green, your app is live at your Render URL.

render.yaml Reference

services:
  - type: web
    name: echo-echo
    runtime: python
    plan: free
    buildCommand: pip install -r requirements-deploy.txt
    startCommand: uvicorn backend.main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: GROQ_API_KEY
        sync: false
      - key: KIEAI_API_KEY
        sync: false
      - key: PUBLIC_BASE_URL
        value: https://consolidated-echoecho.onrender.com
      - key: PYTHON_VERSION
        value: 3.12.6
⚠️

Free tier limitations: The free Render plan spins down after 15 min of inactivity (first request after sleep takes ~30s). Generated audio files live in an ephemeral filesystem and are lost on redeploy. For persistence, attach a Render Disk mounted at /opt/render/project/src/backend/generated.
